# 图片自动分类功能说明

## 功能概述

系统现在支持基于文件名的智能图片分类，无需接入豆包API即可自动识别和分类图片。如果找不到匹配的相册，系统会自动创建新相册。

## 分类逻辑

### 1. 文件名关键词匹配（优先）
系统会检查文件名是否包含预置相册的关键词：

| 相册名称 | 关键词 | 示例文件名 |
|---------|-------|-----------|
| T恤 | T恤, t恤, 短袖, 长袖, 速干衣 | `patagonia_t恤_速干.png` |
| 内衣 | 内衣, 打底 | `icebreaker_美利奴羊毛_内衣.png` |
| 抓绒衣 | 抓绒衣, 抓绒 | `patagonia_R1_抓绒衣.png` |
| 冲锋衣 | 冲锋衣, 硬壳 | `HH_冲锋衣_防水.png` |
| 软壳 | 软壳, 防泼水 | `HH_软壳外套.png` |

### 2. 已知服装类型列表（用于自动提取）
系统内置了以下服装类型关键词，用于从文件名中提取分类名称（按长度从长到短排序）：

**长词（4-6字）**：
- 冲锋衣、抓绒衣、羽绒服、运动鞋、软壳外套

**中词（2-3字）**：
- T恤、内衣、软壳、外套、裤子、卫衣、夹克、背心、短裤、裙子

**说明**：
- 系统会优先匹配长词，确保识别更准确的分类
- 例如：`户外内衣`会匹配`内衣`而不是`户外`
- `软壳外套`会匹配`软壳外套`而不是`软壳`

### 3. 自动创建新相册（未匹配时）
如果文件名不包含任何已知关键词，系统会：

1. **提取分类名称**（按优先级）：
   - **优先级1**：匹配已知服装类型（冲锋衣、抓绒衣、羽绒服、运动鞋、T恤、内衣等）
     - 按长度从长到短排序，优先匹配更长的关键词
     - 例如：`户外内衣`会匹配`内衣`，`软壳外套`会匹配`软壳外套`
   - **优先级2**：使用正则表达式提取中文词语
     - 优先提取4-6个中文字符（如`户外运动鞋`）
     - 其次提取2-3个中文字符（如`运动鞋`）
     - 过滤非分类词汇（破冰者、美利奴、男款女款、图片照片等）
   - **优先级3**：提取第一个英文单词（3-15个字符）
     - 过滤常见非分类词汇

2. **创建新相册**：
   - 相册名称：提取的分类名称
   - 相册ID：自动生成（`album-xxxxxxxx`）
   - 排序号：自动递增
   - 描述：`"自动创建的相册"`
   - 标记为非系统相册（`isSystem: false`）

### 3. 标签提取
系统会从文件名中提取功能特性标签（不包含服装类型）：

- **功能特性**：防风、防水、保暖、透气、速干
- **材质**：美利奴羊毛、棉、聚酯纤维
- **场景**：户外、登山、休闲、专业、运动


## 注意事项

1. **相册名称唯一性**：当前不强制要求相册名称唯一，可能重复创建相同名称的相册
   - 未来优化：在创建前检查是否已存在同名相册

2. **分类名称提取精度**：依赖于文件名的命名规范，建议使用规范的命名格式
   - 推荐格式：`品牌_类型_特性_颜色.png`
   - 例如：`patagonia_T恤_速干_红色.png`

3. **性能考虑**：批量上传时会为每个图片调用分类逻辑，可能需要优化
   - 未来优化：缓存分类结果，减少重复计算

4. **后续AI接入**：当接入豆包API后，会优先使用AI识别结果
   - 如果AI也未能识别，仍然会使用文件名提取逻辑
   - 保证系统的健壮性和扩展性

## 后续优化方向

1. **智能去重**：创建相册前检查是否已存在同名相册
2. **分类置信度**：根据关键词匹配程度设置置信度阈值
3. **批量优化**：批量上传时缓存分类结果，减少重复计算
4. **用户反馈**：允许用户手动调整分类，系统学习用户偏好
5. **AI增强**：接入豆包API后，提高分类准确率
